Lección 1

¿Dónde encontrar problemas?

La mayoría de estudiantes comienzan un proyecto eligiendo un microcontrolador, un sensor o una herramienta de diseño.

Sin embargo, los mejores proyectos comienzan mucho antes: identificando un problema real.

Antes de abrir KiCad, elegir un ESP32 o diseñar una PCB, pregúntate:

Pregunta principal

¿Qué problema vale la pena resolver?

No empieces buscando componentes.

Empieza buscando necesidades.

Lección 2

Problema vs solución

Esto NO es un problema

  • Voy a usar un ESP32.
  • Voy a hacer una PCB.
  • Voy a usar LoRa.

Esto sí es un problema

  • Necesito monitorear temperatura remotamente.
  • Necesito reducir mediciones manuales.
  • Necesito registrar datos automáticamente.

Un problema describe una necesidad.

Una solución describe cómo resolverla.

Lección 3

Hackaday

Hackaday es una de las mejores fuentes para encontrar proyectos de hardware reales.

Qué observar

  • ¿Qué problema intenta resolver?
  • ¿Quién necesita la solución?
  • ¿Qué limitaciones existían?
  • ¿Qué podría mejorarse?

No copies proyectos. Analiza los problemas que originaron esos proyectos.
