Lección 1

¿Qué es un requisito?

Un requisito describe una capacidad, comportamiento o restricción que el sistema debe cumplir. Los requisitos no hablan de componentes específicos. Hablan de resultados esperados.

No es un requisito

Voy a usar un ESP32.

Sí es un requisito

El sistema deberá transmitir datos mediante WiFi.

Lección 2

Requisitos funcionales

Los requisitos funcionales describen lo que el sistema debe hacer.

  • Medir temperatura.
  • Registrar datos.
  • Transmitir información.
  • Mostrar alertas.

Malo

Sistema inteligente.

Bueno

El sistema deberá registrar una medición cada minuto.

Lección 3

Requisitos no funcionales

Describen restricciones y condiciones de operación.

Ejemplos

  • Consumo máximo 500mA
  • Alimentación 24V
  • Temperatura 0-60°C
  • Costo máximo USD 30

Pregunta

¿Una alimentación de 24V es funcional o no funcional?
